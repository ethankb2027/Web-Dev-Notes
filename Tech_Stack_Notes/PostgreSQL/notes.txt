--- 1. Why Use Postgres? ---

  -> Adheres to SQL standards
  -> Excels at concurrency
  -> Superior at avoiding data corruption
  -> Custom data types, operators, and index types
  -> Extensible, scalable, and protects your data

--- 2. What is a Database ---

  -> Data stored in a table of row and columns
  -> To make changes, you send queries/commands and the database will return a response

--- 3. Change Database Theme ---

  -> Typically found under the preferences tab

--- 4. Create a Database ---

  -> Right click the parent database folder and click the add database button

--- 5. Design a Database ---

  -> One table represents one real world object
  -> Columns store one piece of information
  -> Think about how different tables relate to each other
  -> Reduce redundant data

--- 6. Turn Invoice into a Database ---

  -> Best way to practice making a database is to mimic a real world example

--- 7. Make a Table ---

  -> Example table creation:
    CREATE TABLE customer(
      first_name VARCHAR(30) NOT NULL,
      last_name VARCHAR(30) NOT NULL,
      email VARCHAR(60) NOT NULL,
      company VARCHAR(60) NOT NULL,
      street VARCHAR(50) NOT NULL,
      city VARCHAR(40) NOT NULL,
      state CHAR(2) NOT NULL,
      zip SMALLINT NOT NULL,
      phone VARCHAR(20) NOT NULL,
      birth_date DATE NULL,
      sex CHAR(1) NOT NULL,
      date_entered TIMESTAMP NOT NULL,
      id SERIAL PRIMARY KEY
    );

--- 8. Data Types ---

  Character Types:
    -> Char(5): stores up to a max number of 5 characters
    -> Varchar: store any length of characters
    -> Varchar(20): store up to 20 characters
    -> Text: store any length of characters

  Numeric Types:
    -> Serial: whole numbers that also auto increment (always used for column ids)
      -> Smallserial: 1 to 32,767
      -> Serial: 1 to 2,147,483,647
      -> Bigserial: 1 to 9,223,372,036,854,775,807  
    -> Integer: whole numbers only (always used when you don't need a decimal)
      -> Smallint: -32,768 to 32,767
      -> Integer: -2,147,483,648 to 2,147,483,647
      -> Bigint: -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
    -> Floats: numbers with decimals
      -> Decimal: 131,072 whole digits and 16,383 after decimal
      -> Numeric: same as Decimal
      -> Real: -1E37 to 1E37 (6 places of precision)
      -> Double Precision: -1E307 to 1E308 (15 places of precision)
        -> Used when decimal doesn't have to be very precise
      -> Float: same as Double Precision
  
  Boolean Types: 
    -> True: True, 1, t, y, yes, on
    -> False: False, 0, f, n, no, off
    -> Null: null
  
  Date/Time Types:
    -> Date: no matter what's entered, the format will be YYYY-MM-DD
    -> Time:
      -> '1:30:30 PM': time WITHOUT time zone -> 13:30:30
      -> '1:30:30 PM EST': time WITH time zone -> 13:30:30-5:00
      -> 01:30 AM EST -> 01:30-5:00 (UTC Format)
      -> 01:30 PM PST -> 01:30-8:00
      -> 01:30 PM UTC -> 01:30+00:00
    -> Timestamp: 'DEC-21-1974 1:30 PM EST': timestamp WITH time zone -> 1974-12-21 13:30-5:00
    -> Interval: 
      -> Represents a duration of time
      -> '1 day': interval -> 01:00
      -> '1 D 1 H 1 M 1 S': interval -> 01:01:01:01
      -> You can add and subtract intervals

  Other Types:
    -> Currency
    -> Binary
    -> JSON
    -> Range
    -> Geometric
    -> Arrays
    -> XML
    -> UUID

--- 20. Where ---

  -> Displays rows that meet custom criteria

--- 21. Conditional Operators ---

  -> = : equal
  -> < : less than
  -> > : greater than
  -> <= : less than or equal
  -> >= : greater than or equal
  -> <> OR != : not equal

  -> Example conditional select:
    SELECT * 
    FROM sales_item 
    WHERE discount > .15;

--- 22. Logical Operators ---

  -> AND
  -> OR
  -> NOT

  -> Example conditional select with logical operator:
    SELECT time_order_taken, cust_id
    FROM sales_order
    WHERE time_order_taken > '2018-12-01' AND time_order_taken < '2018-12-31';

--- 23. Order By ---

  -> Organizes select query responses by ascending/descending order of certain columns
  
  -> Example select with order by query:
    SELECT *
    FROM sales_item
    WHERE discount > .15
    ORDER BY discount DESC;

--- 24. Limit ---

  -> Limits the number of rows that are returned from a select query

  -> Example select with limit query:
    SELECT *
    FROM sales_item
    WHERE discount > .15
    ORDER BY discount DESC
    LIMIT 5;

--- 25. GROUP BY ---

  -> Combines the results based on selected columns

  -> Example select with group by query:
    SELECT product_id, SUM(price) AS Total
    FROM item
    WHERE product_id=1
    GROUP BY product_id;

--- 26. Distinct ---

  -> Removes duplicates in a select query response

  -> Example select with distinct query:
    SELECT DISTINCT state
    FROM customer
    WHERE state IN ('CA', 'NJ')
    ORDER BY state;

--- 27. Getting Data from Multiple Tables ---

  -> Methods of getting data from multiple tables include Inner Joins, Outer Joins, and Unions

--- 28. Inner Join ---

  -> Example select with inner join query:
    SELECT item_id, price
    FROM item INNER JOIN sales_item
    ON item.id = sales_item.id AND price > 120.00
    ORDER BY item_id;

--- 29. Join 3 Tables ---

  -> Example three table select query:
    SELECT sales_order.id, sales_item.quantity, item.price, (sales_item.quantity * item.price) AS Total
    FROM sales_order
    JOIN sales_item
    ON sales_item.sales_order_id = sales_order.id
    JOIN item
    ON item.id = sales_item.item_id
    ORDER BY sales_order.id;

--- 30. Arithmetic Operators ---

  -> Addition: +
  -> Subtraction: -
  -> Multiplication: *
  -> Division: /
  -> Integer Division: DIV
  -> Modulus: %

--- 31. Join with Where ---

  -> Example of joining via where (not best practice):
    SELECT item_id, price
    FROM item, sales_item
    WHERE item.id = sales_item.item_id
    AND price > 120.00
    ORDER BY item_id;

--- 32. Outer Joins ---

  -> Returns all rows of a table even when a match isn't found
    -> Left Outer Join (preferred)
    -> Right Outer Join

  -> Example select with left outer join query:
    SELECT name, supplier, price
    FROM product LEFT JOIN item
    ON item.product_id = product.id
    ORDER BY name;

--- 33. Cross Joins ---

  -> Includes data from both tables

  -> Example select with cross join query:
    SELECT sales_order_id, quantity, product_id
    FROM item CROSS JOIN sales_item
    ORDER BY sales_order_id

--- 34. Unions ---

  -> Combines the results of two or more select queries
    -> Only works if each result has the same number of columns and data types

--- 35. Extract ---

  -> Returns either the day, month, or year from a date

  -> Example select with extract query:
    SELECT first_name, last_name, street, city, zip, birth_date
    FROM customer
    WHERE EXTRACT (MONTH FROM birth_date) = 12
    UNION
    SELECT first_name, last_name, street, city, zip, birth_date
    FROM sales_person
    WHERE EXTRACT (MONTH FROM birth_date) = 12
    ORDER BY birth_date;

--- 36. IS NULL ---

  -> Used to find if values are null

  -> Example select with null query:
    SELECT product_id, price
    FROM item
    WHERE price IS NOT NULL;

--- 37. SIMILAR LIKE & ~ ---

  -> Returns any string matches

  -> Example select with similar query:
    SELECT first_name, last_name
    FROM customer
    WHERE first_name SIMILAR TO 'M%';

  -> Example select with like query:
    SELECT first_name, last_name
    FROM customer
    WHERE first_name LIKE 'A_____';

  -> Regular Expression Patterns:
    -> . OR _ : any single character
    -> * : 0 or more
    -> + : 1 or more
    -> ^ : beginning of string
    -> $ : end of string
    -> [^ab] : not a & b
    -> [ab] : a & b
    -> [A-Z] : all uppercase letters
    -> [a-z] : all lowercase letters
    -> [0-9] : all numbers
    -> {n} : n instances of
    -> {m,n} : between m & n instances of
    -> m|n : match m or n

  -> Example select with regular expression query:
    SELECT first_name, last_name
    FROM customer
    WHERE last_name ~ 'ez|son';

--- 38. GROUP BY ---

  -> Example select with group by query:
    SELECT EXTRACT (MONTH FROM birth_date) AS Month, COUNT(*) AS Amount
    FROM customer
    GROUP BY Month
    ORDER BY Month;

--- 39. HAVING ---

  -> Narrows results based on a condition

  -> Example select with having query:
    SELECT EXTRACT (MONTH FROM birth_date) AS Month, COUNT(*) AS Amount
    FROM customer
    GROUP BY Month
    HAVING COUNT(*) > 1
    ORDER BY Month;

--- 40. AGGREGATE FUNCTIONS ---

  -> Returns a value from multiple parameters

  -> Example select with aggregate functions query:
    SELECT COUNT(*) AS Items, SUM(price) AS Value, ROUND(AVG(price), 2) AS Avg, MIN(price) AS Min, MAX(price) AS Max
    FROM item;

--- 41. WORKING WITH VIEWS ---

  -> Results of select queries that are stored in the database

  -> Example view creation:
    CREATE VIEW purchase_order_overview AS
    SELECT sales_order.purchase_order_number, customer.company, sales_item.quantity, product.supplier, product.name, item.price,
    (sales_item.quantity * item.price) AS Total, CONCAT(sales_person.first_name, ' ', sales_person.last_name) AS Salesperson
    FROM sales_order
    JOIN sales_item
    ON sales_item.sales_order_id = sales_order.id
    JOIN item
    ON item.id = sales_item.item_id
    JOIN customer
    ON sales_order.cust_id = customer.id
    JOIN product
    ON product.id = item.product_id
    JOIN sales_person
    ON sales_person.id = sales_order.sales_person_id
    ORDER BY purchase_order_number;

  -> Example view deletion:
    DROP VIEW purchase_order_overview;

--- 51. PL/pgSQL ---

--- 52. Variables in Functions ---

  -> Example PL/pgSQL function with variables:
    CREATE OR REPLACE FUNCTION fn_get_random_number(min_val int, max_val int) RETURNS int AS
    $body$
    DECLARE 
      rand int;
    BEGIN
      SELECT random()*(max_val - min_val) + min_val INTO rand;
      RETURN rand;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_get_random_number(1,50);

--- 53. Store Rows in Variables ---

  -> Example PL/pgSQL function with stored rows:
    CREATE OR REPLACE FUNCTION fn_get_random_salesperson() RETURNS varchar AS
    $body$
    DECLARE 
      rand int;
      emp record;
    BEGIN
      SELECT random()*(5 - 1) + 1 INTO rand;
      SELECT *
      FROM sales_person
      INTO emp
      WHERE id = rand;
      RETURN CONCAT(emp.first_name, ' ', emp.last_name);
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_get_random_salesperson();

--- 55. Using Multiple Outs ---

  -> Example PL/pgSQL function with multiple outs:
    CREATE OR REPLACE FUNCTION fn_get_cust_birthday(IN the_month int, OUT bd_month int, OUT bd_day int, OUT f_name varchar, 
                                                    OUT l_name varchar) AS
    $body$
    BEGIN
      SELECT EXTRACT(MONTH FROM birth_date), EXTRACT(DAY FROM birth_date), first_name, last_name
      INTO bd_month, bd_day, f_name, l_name
      FROM customer
      WHERE EXTRACT(MONTH FROM birth_date) = the_month
      LIMIT 1;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_get_cust_birthday(12);

--- 56. Return Query Results ---

  -> Example PL/pgSQL function that returns a query result from one table:
    CREATE OR REPLACE FUNCTION fn_get_sales_people()

    BEGIN
      RETURN QUERY

      FROM sales_person;
    END;
    $body$
    LANGUAGE plpgsql

  -> Example PL/pgSQL function that returns a query result from multiple tables:
    CREATE OR REPLACE FUNCTION fn_get_10_expensive_prods() 
    RETURNS TABLE (
      name varchar,
      supplier varchar,
      price numeric
    ) AS
    $body$
    BEGIN
      RETURN QUERY
      SELECT product.name, product.supplier, item.price
      FROM item
      NATURAL JOIN product
      ORDER BY item.price DESC
      LIMIT 10;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT (fn_get_10_expensive_prods()).*;

--- 57. IF ELSEIF and ELSE ---

  -> Example PL/pgSQL function with IF, ELSEIF, and ELSE:
    CREATE OR REPLACE FUNCTION fn_check_month_orders(the_month int) 
    RETURNS varchar AS
    $body$
    DECLARE 
      total_orders int;
    BEGIN
      SELECT COUNT(purchase_order_number)
      INTO total_orders
      FROM sales_order
      WHERE EXTRACT(MONTH FROM time_order_taken) = the_month;
      IF total_orders > 5 THEN
        RETURN CONCAT(total_orders, ' Orders : Doing Good');
      ELSEIF total_orders < 5 THEN
        RETURN CONCAT(total_orders, ' Orders : Doing Bad');
      ELSE
        RETURN CONCAT(total_orders, ' Orders : On Target');
      END IF;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_check_month_orders(12);

--- 58. CASE Statement ---

  -> Example PL/pgSQL function with a CASE Statement:
    CREATE OR REPLACE FUNCTION fn_check_month_orders(the_month int) 
    RETURNS varchar AS
    $body$
    DECLARE 
      total_orders int;
    BEGIN
      SELECT COUNT(purchase_order_number)
      INTO total_orders
      FROM sales_order
      WHERE EXTRACT(MONTH FROM time_order_taken) = the_month;
      CASE
        WHEN total_orders < 1 THEN
          RETURN CONCAT(total_orders, ' Orders : Terrible');
        WHEN total_orders > 1 AND total_orders < 5 THEN
          RETURN CONCAT(total_orders, ' Orders : On Target');
        ELSE
          RETURN CONCAT(total_orders, ' Orders : Doing Good');
      END CASE;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_check_month_orders(12);

--- 59. Loop Statement ---

  -> Loop Template:
    LOOP
      statements
      EXIT WHEN condition is true;
    END LOOP;

  -> Example PL/pgSQL function with a loop:
    CREATE OR REPLACE FUNCTION fn_loop_test(max_num int) 
    RETURNS int AS
    $body$
    DECLARE 
      j int DEFAULT 1;
      tot_sum int DEFAULT 0;
    BEGIN
      LOOP
        tot_sum := tot_sum + j;
        j := j + 1;
        EXIT WHEN j > max_num;
      END LOOP;
      RETURN tot_sum;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_loop_test(5);

--- 60. FOR LOOP ---

  -> FOR LOOP Template:
    FOR counter_name IN start_value .. end_value BY stepping
    LOOP
      statements
    END LOOP;

  -> Example PL/pgSQL function with a FOR LOOP:
    CREATE OR REPLACE FUNCTION fn_for_test(max_num int) 
    RETURNS int AS
    $body$
    DECLARE 
      tot_sum int DEFAULT 0;
    BEGIN
      FOR i IN 1 .. max_num BY 2
        LOOP
          tot_sum := tot_sum + i;
        END LOOP;
      RETURN tot_sum;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_for_test(5);

  -> Example PL/pgSQL function with a FOR LOOP in reverse:
    CREATE OR REPLACE FUNCTION fn_for_test(max_num int) 
    RETURNS int AS
    $body$
    DECLARE 
      tot_sum int DEFAULT 0;
    BEGIN
      FOR i IN REVERSE max_num .. 1 BY 2
        LOOP
          tot_sum := tot_sum + i;
        END LOOP;
      RETURN tot_sum;
    END;
    $body$
    LANGUAGE plpgsql

    SELECT fn_for_test(5);

--- 61. Result Sets, Blocks & Raise Notice ---

  -> Example DO Block:

      END;
    $body$
    LANGUAGE plpgsql 

--- 62. For Each and Arrays ---

  -> Example For Each with an Array:

      BEGIN

    LANGUAGE plpgsql 

--- 63. While Loop ---

  -> Example While Loop:
    DO
    $body$
      DECLARE
        j int DEFAULT 1;
        tot_sum int DEFAULT 0;
      BEGIN
        WHILE j <= 10
          LOOP
            tot_sum := tot_sum + j;
            j := j + 1;
          END LOOP;
        RAISE NOTICE '%', tot_sum;
      END;
    $body$
    LANGUAGE plpgsql 

--- 64. Continue ---

  -> Jumps to the beginning of a loop

  -> Example loop with Continue:
    DO
    $body$
      DECLARE
        i int DEFAULT 1;
      BEGIN
        LOOP
          i := i + 1;
          EXIT WHEN i > 10;
          CONTINUE WHEN MOD(i, 2) = 0;
          RAISE NOTICE 'Num : %', i;
        END LOOP;
      END;
    $body$
    LANGUAGE plpgsql 

--- 65. Stored Procedures ---

  -> Unlike functions, these can update data

  -> Stored Procedure Template:
    CREATE OR REPLACE PROCEDURE procedure_name(parameters)
    AS
    $body$
    DECLARE
    BEGIN
    END;
    $body$
    LANGUAGE plpgsql;

  -> Example Stored Procedure creation and execution:
    CREATE OR REPLACE PROCEDURE pr_debt_paid(
      past_due_id int,
      payment numeric,
      INOUT msg varchar
    ) AS
    $body$
    DECLARE
    BEGIN
      UPDATE past_due
      SET balance = balance - payment
      WHERE id = past_due_id;
      COMMIT;
    END;
    $body$
    LANGUAGE plpgsql;

    CALL pr_debt_paid(1, 10.00);

--- 66. Triggers ---

  -> When you want an action to automatically execute after an event occurs

  -> Pros of Triggers:
    -> Can be used for auditing/logging
    -> They can be used to validate data
    -> Make certain events always happen
    -> Ensure integrity between different databases
    -> Call functions or procedures
    -> Triggers are recursive
  -> Cons of Triggers:
    -> Triggers add execution overhead
    -> Nested/recursive trigger errors can be hard to debug
    -> Invisible to the client which can cause confusion when actions aren't allowed

  -> Trigger Function Template:
    CREATE FUNCTION trigger_function()
      RETURNS TRIGGER
      LANGUAGE plpgsql
    AS
    $body$
    BEGIN
    END;
    $body$

  -> Trigger Template:
    CREATE TRIGGER trigger_name
      {BEFORE | AFTER} {event} -- Event : insert, update, delete, truncate
    ON table_name
      [FOR [EACH] {ROW | STATEMENT}]
        EXECUTE PROCEDURE trigger_function

  -> Example Trigger function:
    CREATE OR REPLACE FUNCTION fn_log_dist_name_change()
      RETURNS TRIGGER
      LANGUAGE plpgsql
    AS
    $body$
    BEGIN
      IF NEW.name <> OLD.name THEN
        INSERT INTO distributor_audit(dist_id, name, edit_date)
        VALUES (OLD.id, OLD.name, NOW());
      END IF;
      RAISE NOTICE 'Trigger Name : %', TG_NAME;
      RAISE NOTICE 'Table Name : %', TG_TABLE_NAME;
      RAISE NOTICE 'Operation : %', TG_OP;
      RAISE NOTICE 'When Executed : %', TG_WHEN;
      RAISE NOTICE 'Row or Statement : %', TG_LEVEL;
      RAISE NOTICE 'Table Schema : %', TG_TABLE_SCHEMA;
      RETURN NEW;
    END;
    $body$

  -> Example Trigger:
    CREATE TRIGGER tr_dist_name_changed
      BEFORE UPDATE
      ON distributor
      FOR EACH ROW
      EXECUTE PROCEDURE fn_log_dist_name_change();

  -> Example Trigger deletion:
    DROP EVENT TRIGGER tr_block_weekend_changes;

--- 67. Cursors ---

  -> Used to step forward/backward through rows of data and can perform actions on those rows

  -> Example Cursor in a do block:
    DO 
    $body$
      DECLARE
        msg text DEFAULT '';
        rec_customer record;
        cur_customers CURSOR FOR 
          SELECT * FROM customer;
      BEGIN
        OPEN cur_customers;
        LOOP 
          FETCH cur_customers INTO rec_customer;
          EXIT WHEN NOT FOUND;
          msg := msg || rec_customer.first_name || ' ' || rec_customer.last_name || ', ';
        END LOOP;
        RAISE NOTICE 'Customer : %', msg;
      END;
    $body$

  -> Example Cursor in a function:
    CREATE OR REPLACE FUNCTION fn_get_cust_by_state(c_state varchar)
    RETURNS text
    LANGUAGE plpgsql
    AS
    $body$
      DECLARE
        cust_names text DEFAULT '';
        rec_customer record;
        cur_cust_by_state CURSOR (c_state varchar) FOR 
          SELECT first_name, last_name, state
          FROM customer
          WHERE state = c_state;
      BEGIN
        OPEN cur_cust_by_state(c_state);
        LOOP
          FETCH cur_cust_by_state INTO rec_customer;
          EXIT WHEN NOT FOUND;
          cust_names := cust_names || rec_customer.first_name || ' ' || rec_customer.last_name || ', ';
        END LOOP;
        CLOSE cur_cust_by_state;
        RETURN cust_names;
      END;
    $body$
